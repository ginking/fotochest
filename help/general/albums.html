Intalling FotoChest

We are working hard to provide FotoChest with an automated web based installer, but until then, follow these simple steps to get up and working.

Requirements

  • Web server with PHP 5 and the ability to run CodeIgniter
  • CodeIgniter compatable database.
  • GD2 Library. You can use the others supported by CodeIgniter but you need to change it manually.
  • At least 15 mb of memory to process photo uploads.

Database

  • Create a new database to use with FotoChest. Any database supported by CodeIgniter can be used.
  • Run setup.sql which can be found in /assets/sql/setup.sql
  • Update /config/database.php with your database credentials.
  • If you altered the name of your tables (not recommended) you can force the application to point to them by changing the values in /config/settings.php

Configuration

  • In the photoSettings database table modify the absolute file path to point to where FotoChest will be installed. Often times this is from the absolute root of your host.

Run the Installer

  • After uploading your modified files, point your browser to /fotochestroot/installer/install. This creates a user in the database with an account of admin@admin.com and a password of admin. The installer should be deleted immedietly after running.

Troubleshooting

If you have problems check out the forums or post a ticket on github.com
